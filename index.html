Overview

Self-Checkout Machines

  • Objective: This project aims to identify key UX challenges in self-checkout kiosks by analyzing real-world user interactions. The focus is on understanding common pain points such as bagging area errors, slow touchscreen responsiveness, and the need for manual store assistance when purchasing restricted items.
  • Methodology: The study involved conducting on-site observations and unbiased interviews with users actively interacting with self-checkout machines. The process included developing structured, neutral interview questions to gather insights into user expectations, experiences, and frustrations. These findings were then synthesized into user personas and a storyboard to visualize the user journey.
  • Outcome: The project resulted in the creation of detailed user personas representing different types of self-checkout users, such as the efficient/younger shopper and the careful, methodical middle-aged user. Additionally, a storyboard was developed to illustrate the self-checkout experience, highlighting both seamless interactions and moments of user frustration. These insights can be used to improve self-checkout kiosk design for a more user-friendly and efficient experience.

Observations & Interviews

Through on-site observations and interviews, we gathered insights from different users of self-checkout systems
Below are key findings from our user studies.

Key Observations

  • Users faced common issues like bagging area errors and manual assistance for age-restricted items.
  • Speed-focused users tended to make more errors, such as double scanning items.
  • More methodical users carefully checked each step but still preferred self-checkout over cashier lines.
  • Some users required assistance for at least one aspect, such as barcode issues or restricted purchases.

Interview Questions

  • What were your expectations before using self-checkout?
  • Did you encounter any difficulties during the process? If so, what happened?
  • How intuitive do you think the interface was? Any confusing parts?
  • How does this system compare to other self-checkouts you’ve used?
  • What improvements would make this experience better?
  • Did you need any additional assistance? If so, for what?
  • Do you prefer self-checkout or a cashier? Why?

Personas

From the interview and observation, two personas were created as a result

Interface Problems For John

  • Bagging Area Errors: System sometimes gives a popup of a bagging area error message even when items are placed correctly
  • Touchscreen UI Responsiveness: Touchscreen is slightly laggy and processes slow, especially during the payment processing
  • Double Scanning: When scanning items too quickly, double scanning can happen and this requires store associate’s code or card scan to continue
  • Age Restricted Items: If buying medicine, alcohol, or other aged restricted items, need store associate’s verification

Interface Problems For Mike

  • Bagging Area Errors: Does occasionally pop up, but not as much because they are much more steady and careful with each process
  • Age Restricted Items: If buying medicine, alcohol, or other aged restricted items, need store associate’s verification

Why John represents this User Type

  • Prioritizes speed and efficiency, often rushing due to other commitments
  • Tech-comfortable young adult, familiar with self-checkout and prefers automation over human interaction
  • Gets frustrated by inefficiencies like slow UI, bagging errors, or mandatory store assistance
  • More prone to mistakes due to fast scanning but prefers self-checkout over cashiers
  • Represents a large group of users who value speed and are comfortable with technology

Why Mike represents this User Type

  • Middle-aged frequent shopper, steady and careful in his checkout process
  • Comfortable with self-checkout, appreciates its convenience but is not in a rush
  • Takes time to verify items, ensuring accuracy before proceeding
  • Understands self-checkout downsides (e.g., store assistance for certain items) but still prefers it
  • Represents users who have adapted to self-checkout, prioritizing reliability over speed
